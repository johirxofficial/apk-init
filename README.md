## 📖 Overview

`apk-init` is a dual-workflow GitHub Actions toolkit designed to:
1. **Scaffold** a fully structured Android project directly in your repository (`android-init.yml`)
2. **Build, test, and package** your app into downloadable APK artifacts (`build-app.yml`)

Perfect for open-source projects, template repositories, rapid prototyping, and CI/CD education. **100% transparent, zero secrets required.**

---

## ✨ Features

| Feature | Description |
|---------|-------------|
| 🚀 **One-Click Init** | Generates `app/`, `src/main/`, manifests, Gradle files & wrapper automatically |
| 🛠️ **Modern Stack** | AGP `8.1.1`, Java `17`, Compile SDK `33`, Min SDK `24`, AndroidX enabled |
| 📦 **Artifact Export** | Uploads `debug` & `unsigned release` APKs for testing & distribution |
| 🔓 **Open Source Ready** | No keystores, no signing secrets, fully reproducible builds |
| 🔄 **CI/CD Pipeline** | Auto-triggers on push/PR, supports manual dispatch, caches Gradle dependencies |
| 📚 **Extensible** | Easy to add dependencies, permissions, flavors, AABs, or static analysis |

## 🚀 Quick Start

### Option 1: Use as a Template (Recommended)
1. Click **Use this template** → **Create a new repository**
2. Go to **Actions** → Enable workflows
3. Run **Android Init** workflow (manual dispatch)
4. Once complete, run **Build Android App** workflow
5. Download APKs from the **Artifacts** section

### Option 2: Clone & Run
```bash
git clone https://github.com/YOUR_USERNAME/apk-init.git
cd apk-init
# Enable Actions in GitHub UI, then trigger workflows manually
```

## 🔧 Workflow 1: `android-init.yml`

**Purpose:** Automatically generates a complete Android project structure and commits it to your repo.

### 📥 How to Trigger
- **Manual:** `Actions → Android Init → Run workflow`
- **Auto:** Not triggered by default (safety measure). Can be enabled via `push`/`workflow_dispatch`

### 📄 What It Creates
| File/Dir | Purpose |
|----------|---------|
| `settings.gradle` | Includes `:app` module & sets project name |
| `gradle.properties` | Enables AndroidX & Jetifier |
| `build.gradle` (root) | Configures Google/Maven repos & AGP `8.1.1` |
| `app/build.gradle` | Module config, SDK versions, base dependencies |
| `AndroidManifest.xml` | App manifest with `INTERNET` permission |
| `MainActivity.java` | Basic `AppCompatActivity` entry point |
| `activity_main.xml` | Empty `RelativeLayout` ready for UI |
| `gradle/wrapper/` | Gradle `8.1.1` wrapper binaries |

### ⚙️ Customization Points
Edit the YAML before running:
```yaml
# Step 2 variables
PKG="com.yourname.appname"          # Change package/namespace
compileSdk 34                       # Update target SDKs
dependencies {                      # Add libraries
    implementation 'androidx.webkit:webkit:1.9.0'
}
```

## 🏗️ Workflow 2: `build-app.yml`

**Purpose:** Continuous Integration pipeline that builds, lints, tests (optional), and uploads APK artifacts.

### 📥 Triggers
```yaml
on:
  push:
    branches: [ "main", "master" ]
  pull_request:
    branches: [ "main", "master" ]
  workflow_dispatch:  # Manual run
```

### 🔄 Pipeline Flow
```
Checkout → Setup Java 17 → Setup Gradle (Cache) 
    → Verify Wrapper → Run Tests/Lint (continue-on-error) 
    → assembleDebug → assembleRelease 
    → Upload Artifacts
```

### 📦 Artifacts Output
| Name | Path | Retention | Use Case |
|------|------|-----------|----------|
| `WebInApp-debug` | `app/build/outputs/apk/debug/app-debug.apk` | 14 days | Testing, debugging |
| `WebInApp-release-unsigned` | `app/build/outputs/apk/release/app-release-unsigned.apk` | 14 days | Distribution, local signing |

### 🛠️ Local Testing Commands
```bash
./gradlew clean assembleDebug --stacktrace
./gradlew installDebug          # Pushes to connected device/emulator
ls -lh app/build/outputs/apk/debug/
```

## ⚙️ Advanced Customization

### ➕ Add Android App Bundle (AAB)
Append to `build-app.yml`:
```yaml
- name: 📦 Build App Bundle
  run: ./gradlew bundleRelease --stacktrace --no-daemon

- name: 📤 Upload AAB
  uses: actions/upload-artifact@v4
  with:
    name: WebInApp-bundle
    path: app/build/outputs/bundle/release/app-release.aab
```

### 🧹 Enable Product Flavors
Add to `app/build.gradle`:
```gradle
android {
    flavorDimensions "version"
    productFlavors {
        free { dimension "version"; applicationIdSuffix ".free" }
        paid { dimension "version"; applicationIdSuffix ".paid" }
    }
}
```

### 🔍 Add Static Analysis
```yaml
- name: 🔍 Lint Report
  run: ./gradlew lintDebug
  continue-on-error: true

- name: 📄 Upload Lint Report
  uses: actions/upload-artifact@v4
  with:
    name: lint-results
    path: app/build/reports/lint-results-debug.html
```

## 🔐 Open Source & Security Notes

✅ **No Secrets Required**  
This project intentionally avoids signing secrets to remain fully transparent and fork-friendly. Anyone can verify the build process and reproduce APKs.

🔒 **Signing for Production**  
If you plan to publish to Google Play:
1. Generate a keystore locally: `keytool -genkeypair ...`
2. Sign the unsigned APK: `apksigner sign --ks my-release-key.jks app-release-unsigned.apk`
3. **Never** commit keystores or passwords to public repositories.

🌐 **Reproducible Builds**  
All builds use fixed dependency versions and deterministic Gradle configurations. Forks will produce identical APKs when built on the same commit.

## 🛠️ Troubleshooting

| Issue | Solution |
|-------|----------|
| `Workflow not triggering` | Ensure Actions are enabled: `Settings → Actions → General → Allow all actions` |
| `Gradle wrapper missing` | Run `android-init.yml` first, or commit `gradle/wrapper/` manually |
| `Build fails with SDK not found` | AGP 8.1.1 auto-downloads SDK 33. Ensure `compileSdk 33` matches |
| `Artifact not appearing` | Check `path:` matches your `app/build.gradle` output directory |
| `Java version mismatch` | Workflow enforces Java 17. Remove local `JAVA_HOME` overrides if conflicting |
